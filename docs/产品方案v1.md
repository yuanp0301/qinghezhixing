# 知识可视化站点产品方案（v1）

## 1. 产品定位

本项目定位为“知识可视化站点”，用于沉淀与分发可交互的知识内容（如互动动画、交互式课件、可视化讲解页面）。

当前 v1 先聚焦一个菜单模块：

- 菜单：`互动动画管理`
- 目标：让用户可以上传 HTML 互动内容，并生成可分发的唯一下载链接。

后续会持续扩展其他菜单模块，形成完整的知识可视化平台能力。

## 2. v1 目标

在最短周期内打通“上传 -> 存储 -> 生成链接 -> 下载”的闭环，确保：

- 用户可稳定上传 HTML 文件。
- 系统可生成唯一下载 key 并返回下载 URL。
- 下载链路可控，具备“一次性使用”能力与可追踪记录。

## 3. 目标用户与典型场景

- 目标用户：
  - 内容制作人员（制作互动知识内容）
  - 教研/培训人员（分发互动学习材料）
  - 内部运营人员（临时分享可视化页面）

- 典型场景：
  - 上传互动动画 HTML 后，将下载链接发送给指定接收方。
  - 接收方下载后链接失效，避免二次扩散。
  - 运营人员查看 key 使用记录，确认分发是否完成。

## 4. 信息架构（v1）

- 一级导航（当前）：
  - `互动动画管理`

- 二级能力（当前菜单下）：
  - 上传文件
  - 链接生成
  - 下载访问
  - 使用记录

- 扩展原则（后续）：
  - 新菜单可独立增加，不影响 `互动动画管理` 的现有数据与流程。

## 5. 核心功能说明（互动动画管理）

### 5.1 上传 HTML 文件

- 用户输入用户名并选择 `.html` 文件上传。
- 系统对文件进行基础校验（类型、大小）。
- 上传成功后写入文件记录。

### 5.2 生成唯一 key

- key 规则：
  - `username_normalized = trim(lowercase(username))`
  - `raw = username_normalized + ":" + file_id`
  - `key = SHA256(raw)`，输出 64 位小写 hex

- 规则目标：
  - 结果稳定可复算
  - 全局唯一（基于唯一 `file_id`）

### 5.3 生成下载 URL

- 系统返回带 key 的 URL（如 `/d/{key}`）。
- 用户可复制后分发给接收方。

### 5.4 下载与失效机制

- 下载前校验：
  - key 格式正确
  - key 为系统生成且存在
  - key 状态为未使用

- 首次下载成功后：
  - key 状态改为已使用
  - 写入 key 使用记录

- 再次访问同一 URL：
  - 返回“已失效/已使用”

## 6. 页面与交互（v1）

### 6.1 上传页

- 输入项：用户名、HTML 文件。
- 操作：点击上传。
- 反馈：上传中、上传成功、上传失败。

### 6.2 结果页

- 展示项：`file_id`、`key`、下载 URL。
- 操作：一键复制下载链接。

### 6.3 下载结果页

- 成功：触发下载并提示“链接已使用”。
- 失败：明确失败原因（无效 key、未找到、已使用）。

## 7. 业务规则（v1）

- 仅支持 HTML 文件上传。
- 一个 key 只能成功下载一次。
- 下载成功必须落使用记录。
- 系统保存上传与下载行为日志，用于审计与排障。

## 8. 验收标准（v1）

- 可上传 HTML 并完成存储。
- 可生成符合规则的唯一 key。
- 可返回并访问下载 URL。
- 同一 key：首次下载成功，二次下载失败。
- 能查询到对应 key 的使用记录。

## 9. v1 边界与后续规划

- v1 仅交付 `互动动画管理` 菜单的核心闭环，不引入复杂内容编辑能力。
- 后续可扩展方向：
  - 资源库与标签管理
  - 内容模板管理
  - 访问统计与报表
  - 权限与团队协作
